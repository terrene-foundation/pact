# 505: Cross-Team Coordinator Agent Definition

**Milestone**: 5 — Cross-Functional Bridges
**Priority**: Medium (universal agent for every team)
**Estimated effort**: Small

## Description

Define and implement the Cross-Team Coordinator agent — a universal agent present in every team that manages bridge interactions. The Coordinator knows which bridges are active, routes incoming requests to the correct team member, and escalates bridge requests to the Team Lead for approval. This is a specialized agent role, not a new agent platform.

## Tasks

- [ ] Define Cross-Team Coordinator agent definition (COC format):
  - Role: Manage bridge interactions; route cross-team requests
  - Constraint envelope:
    - Operational: may send and receive bridge messages; cannot act on behalf of team without Team Lead approval
    - Data Access: may read bridge-eligible content; cannot access confidential workspace content
    - Communication: bridge channels only; no direct external communication
    - Financial: $0
    - Temporal: active during business hours
  - Allowed tools: bridge messaging, bridge request initiation, incoming message routing
- [ ] Implement Coordinator agent behavior:
  - `CoordinatorAgent.handle_incoming(message)`:
    - Verify message (bridge must be active, sender authorized)
    - Route to appropriate specialist or Team Lead
    - Create audit anchor
  - `CoordinatorAgent.request_bridge(providing_team, purpose)`:
    - Create BridgeRequest
    - Route to Team Lead for approval
    - Track request status
  - `CoordinatorAgent.list_active_bridges()`:
    - Return active bridges and their status
- [ ] Register Coordinator in every team definition:
  - Add to `TeamDefinition.universal_agents` list
  - Every team generated by Organization Builder (M7) includes a Coordinator
- [ ] Write unit tests for Coordinator routing logic

## Acceptance Criteria

- Coordinator agent definition complete with constraint envelope
- Coordinator correctly routes incoming bridge messages
- Bridge requests routed to Team Lead for approval
- Every team has exactly one Coordinator
- Unit tests for routing logic
