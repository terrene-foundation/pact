# 803: Create Contributor Guide

**Milestone**: 8 — Documentation & Developer Experience
**Priority**: Medium (needed for open-source community)
**Estimated effort**: Small
**Depends on**: 101

## Description

Create CONTRIBUTING.md — how external developers can contribute to the CARE Platform.

## Tasks

- [ ] Create `CONTRIBUTING.md`:
  - Development setup (clone, install, test)
  - Code style (ruff, black, mypy)
  - Commit conventions (conventional commits)
  - PR process
  - Test requirements (all changes must include tests)
  - Standards alignment (CARE/EATP/CO — contributions must not violate specs)
  - License: Apache 2.0, CLA if required
  - Code of conduct reference
- [ ] Create `docs/development-setup.md`:
  - Python version requirements
  - Virtual environment setup
  - EATP SDK dependency
  - Running tests
  - Linting and type checking

## Acceptance Criteria

- New contributor can set up development environment from docs
- Contribution process clear
- Standards alignment requirements stated
